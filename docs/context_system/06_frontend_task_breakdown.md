# 前端任务拆解文档 - context_system

> Version: v0.1.0
> Last Updated: 2026-03-13
> Status: Draft

## 1. Phase 1 任务

1. FE-001：搭建 `context/files` 页面骨架与路由。
2. FE-002：实现文件列表与详情展示（分页、筛选、空态）。
3. FE-003：实现上传弹窗与基础表单校验。

## 2. Phase 2 任务

1. FE-004：实现可见性策略弹窗（四种策略 + 条件表单）。
2. FE-005：实现成员候选搜索与多选（`SELECTED_USERS`）。
3. FE-006：实现下线/恢复操作与确认机制。

## 3. Phase 3..N 任务

1. FE-007：实现重建索引操作与任务状态反馈。
2. FE-008：实现文件事件（审计）列表展示。
3. FE-009：完善错误码到 UI 行为映射与统一提示。
4. FE-010：补充可观测埋点（关键操作成功率/失败率）。

## 4. 依赖与执行顺序

1. FE-002 依赖 BE-002/BE-003。
2. FE-003 依赖 BE-001。
3. FE-004/FE-005 依赖 BE-004/BE-006。
4. FE-006 依赖 BE-005。
5. FE-007 依赖 BE-007。
6. FE-008 依赖 BE-008。

推荐顺序：FE-001 -> FE-002/FE-003 -> FE-004/FE-005/FE-006 -> FE-007/FE-008 -> FE-009/FE-010。

## 5. 风险与缓解

1. 可见性配置误解：增加可见范围说明与二次确认。
2. 权限按钮误显示：以后端返回权限为准并在刷新后重算。
3. 大文件上传失败体验差：前端提前做大小/类型提示。
