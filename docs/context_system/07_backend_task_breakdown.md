# 后端任务拆解文档 - context_system

> Version: v0.1.0
> Last Updated: 2026-03-13
> Status: Draft

## 1. Phase 1 任务

1. BE-001：新增 context 核心数据表与 ORM 模型。
2. BE-002：实现文件上传接口与 ingest 任务创建。
3. BE-003：实现文件列表与详情查询（ACL + 状态过滤）。

## 2. Phase 2 任务

1. BE-004：实现可见性更新接口与 ACL entry 事务更新。
2. BE-005：实现文件下线/恢复状态机与事件记录。
3. BE-006：实现候选成员查询接口（组织内搜索）。

## 3. Phase 3..N 任务

1. BE-007：实现重建索引接口与任务冲突检测。
2. BE-008：实现审计事件查询接口。
3. BE-009：在 RAG 检索层接入 ACL 过滤器。
4. BE-010：补齐错误码与可观测指标埋点。

## 4. 依赖与执行顺序

1. BE-001 是基础前置。
2. BE-002/BE-003 依赖 BE-001。
3. BE-004/BE-005/BE-006 依赖 BE-001 与权限中间件。
4. BE-007 依赖 BE-002 与 ingestion 调度能力。
5. BE-009 依赖 BE-001/BE-004/BE-005（需要 ACL 与状态模型）。
6. FE 联调依赖：
- FE-003 <- BE-002
- FE-002 <- BE-003
- FE-004 <- BE-004
- FE-006 <- BE-005
- FE-005 <- BE-006
- FE-007 <- BE-007
- FE-008 <- BE-008

## 5. 风险与缓解

1. ACL 更新与主记录更新不一致：使用单事务。
2. 检索性能下降：增加联合索引与 query profile。
3. 任务重复触发：幂等约束 + 409 冲突返回。
