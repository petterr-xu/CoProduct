# 总体检查清单 - context_system

> Version: v0.1.0
> Last Updated: 2026-03-13
> Status: Draft

## 1. 端到端链路检查

1. 用户上传文件 -> 文件可在管理页可见 -> ingest 成功。
2. 调整可见性后，不同角色/成员看到的文件集合符合策略。
3. 文件下线后，预审链路不再返回该文件证据。
4. 文件恢复后，可重新参与检索。
5. 重建索引成功后，检索结果可恢复正常。

## 2. 前后端契约兼容性检查

1. FE/BE 路径、方法、字段、枚举完全一致。
2. 错误码在 FE 有对应 UI 行为。
3. 新增字段不会破坏旧前端解析。

## 3. 阶段完成度检查（Phase 1..N）

1. Phase 1：基础数据模型和上传/查询闭环完成。
2. Phase 2：可见性 + 下线能力完成。
3. Phase 3：重建索引 + 审计 + RAG ACL 集成完成。
4. Phase 4：关键指标与回滚策略可执行。

## 4. 发布 Gate

1. 关键安全项通过：ACL 强制过滤、默认拒绝。
2. 关键可靠性项通过：重建冲突处理、任务状态可追踪。
3. 回滚方案可执行：可关闭 `ALL_USERS`、可关闭 context 检索入口。
